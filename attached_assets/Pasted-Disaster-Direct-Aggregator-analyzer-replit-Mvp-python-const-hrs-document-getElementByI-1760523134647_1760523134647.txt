Made it happen. Your Disaster Direct app now supports:

1) Drawn zones on the map (Leaflet Draw)

New toolbar on the map lets you draw polygons or rectangles.

Click Draw Zone, sketch, then click Save Zone (name it).

Zones render as dashed outlines; they also populate the Scheduler select.

Endpoints:

POST /zones?name=... (body: GeoJSON from the drawing)

GET /zones (list)

DELETE /zones/{id}

GET /zones.geojson (for the map)

GET /zones/centroid/{id} → returns { lat, lon, suggested_radius_km }

2) Per-schedule destinations (webhook/SMS overrides)

Scheduler form now has:

Zone dropdown (optional) – runs will target that zone’s centroid with a suggested radius.

Webhook override – use a different URL just for this schedule.

SMS list – comma-separated numbers (e.g., +15555550123, +15555550124) to override global crew phones.

Storage/schema updated; /cron/run now:

Resolves zone → centroid + radius when set,

Saves a Run snapshot,

If risk_score ≥ threshold, sends to the per-schedule webhook and/or per-schedule SMS list (falls back to global webhook/SMS if overrides aren’t provided).

How to use it fast

Open / → click Draw Zone → use the polygon/rectangle tool → Save Zone.

In Daily Scheduler, pick that zone, set threshold, and optionally paste a webhook/SMS list.

Trigger everything via:

GET /cron/run


(Aim your external scheduler at that URL.)
When a job breaches threshold, the app notifies your chosen destinations.

Notes

I added Leaflet.draw via CDN and wired the draw layer.

The app will approximate a query radius from zone area for aggregation (you still get parcel-level intersections for impact).

SMS sending now accepts override recipients per job.
