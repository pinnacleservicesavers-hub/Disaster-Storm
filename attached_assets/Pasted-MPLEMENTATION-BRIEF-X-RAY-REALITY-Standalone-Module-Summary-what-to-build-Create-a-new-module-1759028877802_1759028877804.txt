MPLEMENTATION BRIEF — X-RAY REALITY (Standalone Module)
Summary (what to build)

Create a new module X-RAY REALITY at /client/src/modules/xray-reality/ that lets users watch storms “live” and annotate the real world with AR. It must work on phones, laptops/desktops, and headsets with a built-in voice guide.

Must-have features (non-negotiable)

Live radar & GOES drawn into a Three.js scene and auto-refreshed (feels live).

Radar refresh: every 30–60s

GOES refresh: every 60–120s

Use cache-busting ?_=${Date.now()} to avoid stale images.

Live Alerts Ticker (NWS active alerts) across the top, refresh every 60s.

3×3 Traffic Cam grids with a state switcher (FL / GA / AL) at the top.

Storm Chasers: 3+ YouTube Live embeds (IDs configurable).

Drone Feeds: Accept HLS (.m3u8) URLs (we will supply), autoplay muted.

Ocean View: Sea-surface temp + waves as auto-refresh raster planes.

AR Measure & Mark:

Drop hazard markers

Draw cut lines

Measure distance/diameter (UI stub ok; visuals update in scene)

Voice guide (Web Speech): Buttons “Voice Tour”, “What is this?”, “How to use”, “Accuracy tips” that speak scripted lines.

Key components (what to implement)
/client/src/ar/useARSupport.ts

Detect WebXR; fall back to “inline” (webcam) if not available.

/client/src/ar/arLayers.ts

Provide addAutoRefreshingRasterPlane(scene, id, urlBuilder, opts) that:

Creates a textured plane (width/height from opts)

Fetches a new texture on an interval (refreshMs)

Appends cache-buster to URL

Returns a handle so we can clean up on unmount

Provide simple addGeoMarker(scene, lat, lng, alt) and addCutLine(scene, coords[]).

/client/src/ar/UniversalAR.tsx

Bootstrap Three.js scene; show title + mode (immersive-ar/inline).

Accept overlays props to draw markers and lines.

This is reused in Measure & Mark and can be embedded elsewhere.

Live Storm View

Build a local Three scene (or reuse UniversalAR) and add two planes:

Radar plane → VITE_RADAR_LATEST_URL

GOES plane → VITE_GOES_LATEST_URL

Both auto-refresh (intervals above).

Show the AlertTicker on top.

Traffic Cams

State selector (FL/GA/AL) → renders a 3×3 grid using env URLs.

Support types: youtube, hls, img (use <iframe>, <video>, <img>).

Storm Chasers

3 tiles using VITE_CHASER_* YouTube embed URLs (autoplay muted).

Drone Feeds

3 tiles using VITE_DRONE_* HLS URLs (<video autoPlay muted controls>).

Do not break if a feed is offline; show “Feed unavailable” placeholder.

Ocean View

Two auto-refreshed planes (SST and Waves) using the ocean env URLs.

AR Measure & Mark

Reuse UniversalAR + three buttons: Add Marker, Draw Cut Line, Measure.

For now, place a marker at a fixed demo coordinate and draw a short demo line; wire real geo later.

UX rules

Default open tab: Live Storm View.

Buttons: rounded, bold labels, high contrast.

If a URL is missing, show a friendly placeholder tile (no crashes).

On mobile, prompt for camera permission once (don’t spam).

All auto-refresh intervals clear on unmount to avoid leaks.
