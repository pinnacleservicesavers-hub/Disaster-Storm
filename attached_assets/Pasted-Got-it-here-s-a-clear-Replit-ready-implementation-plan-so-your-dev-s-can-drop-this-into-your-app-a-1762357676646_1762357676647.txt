4) AI routing & outreach (what actually runs)

Flow on new/updated lead

Classify damage & expand services (tree + roof + fence + pool + windows + water + etc. as needed).

Score urgency + compute routing shortlist per category (Tier 1 → Tier 2).

Outreach sequence fires (phone, text, email) from local area-code numbers.

Keep lead open until all service categories show “✅ contractor assigned” or “not needed”.

How to ensure local numbers

In Twilio: buy/assign local numbers per area code you operate in.

Use Twilio Geo-Match (or your own map: zip → preferred number) to originate calls/SMS from the same area code as the lead.

Caller ID name:

Mobile caller ID is controlled by carriers (you cannot force “Disaster Direct” everywhere).

Use Twilio Branded Calls for supported carriers/Android to show Disaster Direct + logo.

For landlines, you can register CNAM so “Disaster Direct” appears where supported.

5) Lead assignment logic (not “open grab”)

Best practice: smart, fair, and fast—no chaos.

Tier 1 round (10–15 min): Notify top-scored, compliant contractors (specialization, proximity, availability, performance).

If no accept, Tier 2 round.

First accept → locks that service category only (e.g., Tree Service = assigned) while other categories remain open (Roofing, Windows, Pool, etc.).

Each service category shows a checkbox:

tree_assigned, roof_assigned, fence_assigned, pool_assigned, windows_assigned, water_mitigation_assigned, gc_assigned, etc.

The lead closes only when all required categories are assigned & completed (or explicitly marked “not needed”).

6) Sample API (drop into server.js)
import express from "express";
import bodyParser from "body-parser";
import { createLead, expandServices, routeLead, sendOutreach } from "./services/lead.js";

const app = express();
app.use(bodyParser.json());

// 1) Intake webhook (from AI detector, form, or CSV import)
app.post("/api/leads/intake", async (req, res) => {
  const lead = await createLead(req.body);
  await expandServices(lead);        // adds fence/pool/windows/etc as needed
  await routeLead(lead.id);          // precomputes Tier 1/Tier 2 per category
  await sendOutreach(lead.id);       // phone + sms + email
  res.json({ ok: true, leadId: lead.id });
});

// 2) Contractor accepts an assignment
app.post("/api/assignments/:id/accept", async (req, res) => {
  // validate contractor, mark service category as assigned, cancel other offers
  // return updated lead + service state
  res.json({ ok: true });
});

// 3) Status for frontend boards/analytics
app.get("/api/leads", async (req, res) => {
  // return list with ai_confidence, priority, per-category assigned flags
  res.json([]);
});

app.listen(3000);
