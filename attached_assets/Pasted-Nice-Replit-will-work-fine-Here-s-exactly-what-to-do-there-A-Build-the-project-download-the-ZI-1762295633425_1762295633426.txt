Nice—Replit will work fine. Here’s exactly what to do there.

A) Build the project + download the ZIP (your immediate ask)

In your Replit (Python) repl, create a new file named build_storm_suite_v3.py and paste in the full script I gave you.

Open the Shell (bottom pane) and run:

python3 build_storm_suite_v3.py


You’ll see a new file in the left file tree:

storm-disaster-full-suite-v3.zip


Download it from Replit:

Hover over storm-disaster-full-suite-v3.zip in the file tree → click the ⋯ (three dots) → Download.

If you don’t see the dots, you can also zip and export from the Shell:

# (already built) just ensure it's there
ls -lh storm-disaster-full-suite-v3.zip


Then click it in the left panel to trigger the download.

B) Run it right on Replit (backend now, frontend in a second repl)
Backend (FastAPI)

In the Shell, unzip and install:

unzip -o storm-disaster-full-suite-v3.zip -d .
cd storm-disaster-full-suite-v3/backend
pip install -r requirements.txt


Start the API (Replit needs 0.0.0.0 host):

python -m uvicorn app.main:app --host 0.0.0.0 --port 8000


Replit will show a webview link. Append /health to test:

https://<your-repl-domain>.repl.co/health


You should get {"ok": true, ... }.

SMTP/Twilio env (optional):

In Replit → Secrets (lock icon), add:

TWILIO_ACCOUNT_SID, TWILIO_AUTH_TOKEN, TWILIO_FROM

Then re-run the server.

Frontend (Next.js)

Replit typically runs one process per repl. Easiest: make a second Replit (Node.js) for the frontend.

In your local machine, unzip the ZIP and upload the frontend/ folder to a new Replit (Node.js) project
— or in your current repl, copy the folder out of the zip via Shell:

# If you want to try in the same repl (you can’t run both at once here)
cd ../frontend
npm i
echo "NEXT_PUBLIC_API_BASE=https://<your-backend-repl-domain>.repl.co" > .env.local
npm run dev -- -p 3000


Open the port Replit shows for the frontend. You should see the app with links like:

/admin/settings/smtp

/admin/carriers

/admin/legal/templates

/contractor/claims/submit

/legal

/admin/sms

C) Quick sanity checks

SMTP: go to /admin/settings/smtp, enter a test SMTP (use a test inbox), Save.

Carriers: /admin/carriers → add one with an inbox email.

Claim: /contractor/claims/submit → Import a CSV (desc,qty,price), optionally upload Xactimate CSV, click Download Packet, then Auto-post.

Legal: /legal → click Apply Preload, optionally mark counsel-approved.

Templates: /admin/legal/templates → edit/approve/clone versions.

SMS: /admin/sms (requires Twilio env to be set) → send a test.
