Perfect — here are ready-to-paste edge cache rules for Cloudflare, Fastly, and CloudFront (plus an Nginx fallback). They’re tuned for your HMAC-signed tiles/legend so the CDN can cache safely, while bypassing cache for JSON endpoints like /api/impact and /api/impact/geojson.

Cloudflare (Rules + optional Worker)
A) Cache Rules (no code, fast setup)

What it does

Cache images under /api/impact/tiles/* and /api/legend.*

Honor your ttl query param for tiles

Ignore cookies/Authorization (HMAC in URL is the gate)

Bypass cache for JSON endpoints

Create two Cache Rules (Ruleset → Cache Rules):

Bypass JSON

If: http.request.uri.path contains "/api/impact" OR "/api/pollen" OR "/api/incidents"

Then: Cache status = Bypass

Cache tiles & legend

If: (http.request.uri.path contains "/api/impact/tiles/") OR (http.request.uri.path starts_with "/api/legend")

Then:

Cache eligibility: Eligible

Edge TTL: Expression → use int(http.request.uri.query["ttl"]) default 180

Browser TTL: 60

Respect origin Cache-Control: Off (override)

Origin cache key: Include full query string (VERY IMPORTANT)

Bypass cache on Cookie / Authorization: Off (i.e., ignore)

That’s it. If you want stale-while-revalidate, set Browser TTL small (e.g., 30–60s) and keep Edge TTL higher.

B) Optional Worker (fine-grained control)
// wrangler.toml -> routes = ["your-domain.com/*"]
export default {
  async fetch(req, env, ctx) {
    const url = new URL(req.url);

    // Bypass JSON
    if (url.pathname.startsWith("/api/impact") || url.pathname.startsWith("/api/pollen") || url.pathname.startsWith("/api/incidents")) {
      return fetch(req, { cf: { cacheTtl: 0, cacheEverything: false }});
    }

    // Tiles & legend
    if (url.pathname.includes("/api/impact/tiles/") || url.pathname.startsWith("/api/legend")) {
      // Parse TTL from query or default
      const ttl = Math.max(10, Math.min(600, parseInt(url.searchParams.get("ttl") || "180", 10)));
      // Full-URL key (includes sig + query) — great for HMAC
      const cacheKey = new Request(url.toString(), req);

      // Try edge cache
      const cache = caches.default;
      let res = await cache.match(cacheKey);
      if (res) return res;

      // Fetch from origin
      res = await fetch(req, { cf: { cacheTtl: ttl, cacheEverything: true } });
      // Add cache headers (optional, cosmetic)
      const newRes = new Response(res.body, res);
      newRes.headers.set("Cache-Control", `public, max-age=${ttl}, stale-while-revalidate=60`);
      newRes.headers.delete("Set-Cookie");
      await cache.put(cacheKey, newRes.clone());
      return newRes;
    }

    // default
    return fetch(req);
  }
};

AWS CloudFront (Behaviors + Lambda@Edge optional)

Behaviors

Default behavior → your origin (Node app)

/api/impact/tiles* behavior:

Cache Policy: create one that whitelists all query strings (Forward all, cache based on all)

Header/Cookie forwarding: None (unless your origin demands it)

TTL: set Minimum=10, Default=180, Maximum=600

/api/legend* behavior: same cache policy (queries matter).

/api/impact* and /api/pollen* behavior:

Cache Policy: no caching (Minimum/Default/Maximum = 0) or use Managed “CachingDisabled”.

Optional Lambda@Edge (Viewer Response) to enforce headers:

export const handler = async (event) => {
  const r = event.Records[0].cf.response;
  const req = event.Records[0].cf.request;
  const path = req.uri || "";

  if (path.startsWith("/api/impact/tiles/") || path.startsWith("/api/legend")) {
    const ttl = Math.max(10, Math.min(600, parseInt((req.querystring.match(/(?:^|&)ttl=(\d+)/)||[])[1] || "180",10)));
    r.headers['cache-control'] = [{ key: 'Cache-Control', value: `public, max-age=${ttl}, stale-while-revalidate=60` }];
    delete r.headers['set-cookie'];
  }
  return r;
};

Nginx (self-hosted edge/proxy)
# Bypass JSON
location ~ ^/api/(impact|pollen|incidents) {
  proxy_pass http://app:3001;
  proxy_set_header Host $host;
  proxy_set_header X-Forwarded-For $remote_addr;
  add_header Cache-Control "no-store";
}

# Cache tiles/legend by full query
location ~ ^/api/(impact/tiles/|legend) {
  # proxy cache (define a cache zone elsewhere)
  proxy_cache dd_tiles;
  proxy_cache_key "$scheme$proxy_host$request_uri";  # includes full query (and HMAC sig)
  proxy_cache_valid 200 302 10s;
  proxy_cache_valid 404 1m;
  proxy_ignore_headers Set-Cookie;
  add_header X-Cache-Status $upstream_cache_status;

  # TTL override from ?ttl= (fallback 180)
  set $ttl 180;
  if ($arg_ttl) { set $ttl $arg_ttl; }
  add_header Cache-Control "public, max-age=$ttl, stale-while-revalidate=60";

  proxy_pass http://app:3001;
  proxy_set_header Host $host;
}

Purging / Invalidation

Cloudflare: Purge by URL (the signed URL string). If you need wider purges, adopt a short TTL (e.g., 180s) + stale-while-revalidate=60.

Fastly: Purge by URL or use a Surrogate-Key per scheme/grid to purge groups. You can add a header in origin responses: Surrogate-Key: dd-tiles scheme-viridis grid-3.

CloudFront: Invalidate by path (can include query wildcards only via * on path; queries aren’t matched). Prefer short TTLs + versioned query params if you need broad purge.

Sanity checklist (for any CDN)

✅ Key on full query string (HMAC signature + params define uniqueness).

✅ Ignore cookies/Authorization for tiles/legend (access is in the signed URL).

✅ Bypass JSON (/api/impact, /api/geojson, /api/pollen, /api/incidents).

✅ Set/override Cache-Control at the edge if origin can’t.

✅ Honor ttl query for tiles (with safe clamp 10–600s).

✅ Add stale-while-revalidate window (e.g., 60s) for smooth reloads.
