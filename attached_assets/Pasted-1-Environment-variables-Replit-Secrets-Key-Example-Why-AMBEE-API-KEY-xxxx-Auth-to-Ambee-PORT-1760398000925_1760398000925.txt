1) Environment variables (Replit → Secrets)
Key	Example	Why
AMBEE_API_KEY	xxxx...	Auth to Ambee

CORS_ORIGINS	https://your-frontend.example.com,https://*.replit.app	Lock down browser origins
DD_BEARER_TOKEN (optional)	superlongrandom	Static bearer gate (layered with JWT)

CACHE_TTL_MS	60000	Default cache TTL (ms)
COORD_SNAP_DEG	0.01	Snap grid for caching

You already wired most of this in previous steps; these names align with the features we added (auth, rate-limit, caching, JWT).

2) Recommended TTLs by zoom (tiles / impact)

Tiles/Legend TTL (seconds)

z ≤ 7: 600–900 (regional views; slower-changing)

8–10: 180–300

11–12: 120–180

≥ 13 (street): 60–120

/impact & /geojson TTL (server cache)

Default: 60s (increase to 180s during spikes or set adaptively on 429)

Adaptive tweaks

If Ambee returns 429 → temporarily double TTL (surge mode).

If breaker is open → serve last-known cache (stale-while-revalidate feel).

3) Alerts & thresholds (Prometheus or any APM)

Track these metrics (you can expose /metrics if you’re using prom-client):

Error rate (5m): alert at >3% 5xx or >2% 429 sustained 5m

P95 latency: /impact > 1500 ms (5m)

Circuit breaker open: any non-zero for > 2m

Cache hit rate: tiles < 60% (zoom-weighted) → investigate snapping/TTL

Auth failures: > 50 401/403 in 5m → misconfig or attack

Upstream health: consecutive upstream failures > CIRCUIT_THRESHOLD (we open breaker, alert anyway)

4) Health & readiness endpoints

GET /healthz → { ok:true, uptime, version, gitSha }

GET /readyz → includes a cheap Ambee probe and breaker state:
{ ready:true, ambee:"ok", breaker:"closed" }

Use /readyz for rollout gates; /healthz for container liveness.

5) Coordinate snapping (cost saver)

What it does: rounds incoming lat/lng to e.g. 0.01° before caching/fetching.
Effect: 10–50× more cache hits for dense map traffic.
Suggested: start at 0.01° (~1.1 km), dial down to 0.005° if you need more precision.

6) CDN + browser caching

Serve tiles/legend with Cache-Control: public, max-age=<ttl>, stale-while-revalidate=60.

Add ETag to /impact and /impact/geojson; honor If-None-Match to return 304.

7) Logging (structured)

Per request log fields:

requestId, route, method, lat, lng (rounded), status, durationMs, cache: hit|miss, upstreamStatus, breakerState, clientIp, sub (JWT subject if present).

Why: instant debug + easy dashboards.

8) Security hardening

JWT (JWKS) with scopes per route (we added this).

CORS exact origins; no *.

HMAC tile URLs (optional): sign {z,x,y,ttl,pollen,grid,scheme} with short expiry to prevent hotlinking.

Audit log: nightly export minimal fields (time, route, scope, sub, geo hash).

10) Smoke test (copy-paste)

Create a file smoke.sh in your Replit shell (adjust host as needed):

#!/usr/bin/env bash
set -euo pipefail

HOST="${HOST:-http://localhost:3001}"
LAT="${LAT:-33.749}"
LNG="${LNG:--84.388}"

auth_header=()
[[ -n "${DD_BEARER_TOKEN:-}" ]] && auth_header+=( -H "Authorization: Bearer ${DD_BEARER_TOKEN}" )
[[ -n "${DD_INTERNAL_KEY:-}" ]] && auth_header+=( -H "x-api-key: ${DD_INTERNAL_KEY}" )

echo "== /healthz"
curl -sf "${HOST}/healthz" | jq .

echo "== /readyz"
curl -sf "${HOST}/readyz" | jq .

echo "== /impact"
curl -sf "${HOST}/api/impact?lat=${LAT}&lng=${LNG}" "${auth_header[@]}" | jq '.impactScore, .components'

echo "== /impact (pollen)"
curl -sf "${HOST}/api/impact?lat=${LAT}&lng=${LNG}&pollen=1" "${auth_header[@]}" | jq '.impactScore, .components.pollen'

echo "== /pollen"
curl -sf "${HOST}/api/pollen?lat=${LAT}&lng=${LNG}" "${auth_header[@]}" | jq '.risk, .tree, .grass, .weed'

echo "== /impact/geojson"
curl -sf "${HOST}/api/impact/geojson?lat=${LAT}&lng=${LNG}&pollen=1" "${auth_header[@]}" | jq '.features[0].properties.impactScore'

echo "== /legend.png"
curl -sfI "${HOST}/api/legend.png?scheme=traffic&width=256&height=48&bg=solid" | grep -E "200|Content-Type"

echo "== tile sample"
curl -sfI "${HOST}/api/impact/tiles/10/293/394.png?pollen=1&grid=3&scheme=viridis&ttl=180" | grep -E "200|Content-Type"

echo "== OpenAPI"
curl -sf "${HOST}/api/openapi.json" | jq '.info.title, .paths | length'

echo "OK ✅"


Run:

chmod +x smoke.sh
./smoke.sh

11) Rollout plan (safe deploy)

Stage on a non-prod Replit project with all env vars.

Run ./smoke.sh → must pass.

Open /api/docs (Swagger UI) and try each call with auth headers.

Turn on Prometheus scraping (or your APM) and sanity-check metrics.

Set CDN rules for /api/impact/tiles and /api/legend (TTL by query param).

Canary: route 5–10% traffic for 30–60 min; watch error rate, P95 latency, breaker opens.

Roll to 100%.

12) Failure playbook

Spike in 429:

Confirm coordinate snapping active; increase tile TTL 2×; reduce grid to 1; enable surge TTL.

Breaker open:

Verify Ambee status; keep serving cached tiles; shorten UI polling; show “Degraded data” banner.

High latency:

Check CDN cache hit; increase snapping; turn off labels on tiles; prewarm hot incident areas.

Auth 401/403 storm:

Confirm JWKS reachable; issuer/audience exact; scope claim name matches token.

13) What this buys you (quick ROI)

Reliability: retries, breaker, health/ready keep you up when providers wobble.

Security: JWT/JWKS, scopes, CORS, (optional) HMAC tiles keep data controlled.

Cost control: snapping + CDN + adaptive TTL = far fewer Ambee calls.

Speed: cached tiles + ETags → snappy maps and dashboards.

Ops clarity: metrics, structured logs, alerts, and Swagger docs = low friction.
